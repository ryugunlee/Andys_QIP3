# QUANT2 — QIP3 5요인 점수 체계

이 문서는 **QIP3 점수 체계**의 방법론과 팩터 구성, 그리고 그 근거를 정리한다.
기존 점수 체계(`QUANT.md`의 Vscore/Mscore/Fscore/Finalscore/goodstock)는 그대로 두고
**병행하는 두 번째 선별 엔진**으로 추가된 것이다. 두 체계는 서로를 대체하지 않는다.

- 구현 위치: `analysis/qip3_factors.py`·`qip3_weights.py`·`qip3_composites.py`·`qip3_pipeline.py`,
  선별은 `storage/qip3_selection.py`의 `get_goodstock2`.
- 점수 엔진(퍼센타일/스탠다드)·모집단(전체/섹터/산업)·컬럼 네이밍 규칙은 `QUANT.md` 2절과 동일하다.

---

## 0. 설계 원칙

1. **규모 중립성.** 시가총액이나 절대 금액이 아니라, 기업 규모와 무관하게 비교 가능한
   **비율형 지표**만 쓴다(재고자산회전율, 마진, 부채/자본 비율, 수익률 등). 매출·순이익 같은
   절대 금액 팩터는 QIP3 어떤 요인에도 넣지 않는다.
2. **2단 선별(나쁜 기업 배제 → 좋은 기업 순위).** 먼저 생존 리스크가 큰 기업(안정성 하위 20%)을
   무조건 제외하고, 남은 기업만 종합점수로 줄 세운다. Greenblatt의 "좋은 기업을 싼 값에"라는
   2단 구조를 따른다.
3. **가치·성장·모멘텀 우선, 재무건전성 부차.** 사용자 요구에 따라 종합점수 가중치를
   가치 0.35 / 성장 0.25 / 모멘텀 0.25 / 재무건전성 0.15로 둔다. 안정성은 종합점수에 넣지 않고
   **하드 필터 전용**으로 쓴다.
4. **시장 전체·섹터 내 점수 병행 산출.** 모든 요인·종합점수를 통화권(시장) 전체 모집단과
   섹터/산업 내 모집단에서 각각 계산한다(`QIP3 Value` vs `QIP3 ValueSec`/`QIP3 ValueInd`).
   섹터 쏠림은 선별 결과 화면(주식 분석 페이지)에서 확인할 수 있다.

## 1. 근거 연구

QIP3의 팩터 선택과 결합 방식은 검증된 학술 연구·투자 구루의 방법론에 기반한다.

| 요인 | 근거 | 핵심 논지 |
|---|---|---|
| 가치성 | **O'Shaughnessy, _What Works on Wall Street_ (4판)** — Value Composite | PER·PBR·PSR·EV/EBITDA·PCR·주주환원수익률을 **동일가중 합성**한 밸류 컴포지트가 단일 지표보다 더 높고 안정적인 수익. 45년 백테스트에서 최저평가 십분위가 시장을 크게 상회. |
| 성장의 질 | **Novy-Marx (2013), _The Other Side of Value: The Gross Profitability Premium_** | 매출총이익/총자산(GP/A)이 손익계산서 하단 지표(순이익 등)보다 잡음이 적어 미래 수익성을 더 잘 예측. 19개 선진국에서 프리미엄 확인. |
| 모멘텀 | **Jegadeesh & Titman (1993)** | 최근 12개월 수익률(단기 반전 회피를 위해 **최근 1개월 제외**)로 승자를 사는 12-1 모멘텀. 표준 구현은 상위 십분위 매수. |
| 재무건전성 | **Piotroski (2000), F-Score** | 수익성·현금흐름의 질·레버리지 추세·효율성 9개 신호로 "싸고 개선되는" 기업과 "싸고 썩는" 기업을 가른다. 고득점주가 저득점주를 연 20%대 상회(1976–1996). |
| 안정성 | **Altman (1968), Z-Score** | 레버리지·유동성·수익성·시장가치 비율로 부도 위험을 예측(1년 전 80~90% 정확도). 가치 함정(싸 보이지만 부실한 기업) 회피용 방어 스크린. |
| 결합 구조 | **Greenblatt, _Magic Formula_ / Novy-Marx** | 품질(수익성)과 가치를 **각각 순위화해 결합**하면 단일 요인보다 우수. "좋은 기업을 싼 값에." |

> 참고 링크: [What Works on Wall Street 해설](https://www.validea.com/james-p-oshaughnessy) ·
> [Gross Profitability Premium](https://www.ivey.uwo.ca/media/3775548/novy-marx.pdf) ·
> [Jegadeesh-Titman (1993)](https://www.bauer.uh.edu/rsusmel/phd/jegadeesh-titman93.pdf) ·
> [Piotroski F-Score 백테스트](https://www.quant-investing.com/blog/piotroski-f-score-back-test) ·
> [Altman Z-Score](https://en.wikipedia.org/wiki/Altman_Z-score)

이 지표들은 미래 수익을 **보장**하지 않는다. 시장·기간·구현 방식에 따라 성과가 달라지며,
QIP3는 매수·매도를 권유하지 않고 판단을 돕는 정량 자료를 제공할 뿐이다.

## 2. 5요인 정의

각 요인 점수는 구성 팩터의 **점수(0~100)를 가중평균**한 값이라, 요인 점수도 0~100이다.
방향성(HIGHER/RECIPROCAL/NEGATED)과 점수 엔진은 `QUANT.md` 2절과 동일하다.
`(반전)`은 기존 파이프라인에 "클수록 좋다"로 등록된 팩터를 `100 − 점수`로 뒤집어 쓴다는 뜻이다.

### ① 가치성 (Value) — 가중 0.35
O'Shaughnessy Value Composite 방식. 6개 컴포넌트 **동일가중(각 1/6)**.

| 컴포넌트 | 방향 | 비고 |
|---|---|---|
| PER, PBR, PSR, PCR, EV/EBITDA | 낮을수록 좋음 | 각 1/6 |
| 주주환원수익률 = (배당수익률 + 자사주매입수익률)/2 | 높을수록 좋음 | 1/6 |

- PFCR은 잉여현금흐름이 음수일 때 분모 부호가 뒤집혀 순위를 오염시키므로(PROBLEMS #32) 제외하고 PCR을 쓴다.
- PEGR은 가치·성장이 섞인 지표라 이중계상을 피해 Value에서 제외(성장은 성장성 요인에서 따로 본다).
- 내부자/기관 지분 계열은 네이버(한국) 구조적 결측이라(PROBLEMS #9) QIP3 전체에서 미사용.

### ② 성장성 (Growth) — 가중 0.25
| 팩터 | 방향 | 가중 |
|---|---|---|
| EPS 성장률 | 높을수록 좋음 | 0.35 |
| 매출 성장률 | 높을수록 좋음 | 0.35 |
| GP/A (매출총이익/총자산) | 높을수록 좋음 | 0.30 |

- 성장률 2종이 중심, GP/A(Novy-Marx)가 "성장의 질" 앵커.

### ③ 모멘텀 (Momentum) — 가중 0.25
| 팩터 | 방향 | 가중 |
|---|---|---|
| 12-1 모멘텀 (최근 1개월 제외 12개월 수익률) | 높을수록 좋음 | 0.50 |
| 6개월 수익률 | 높을수록 좋음 | 0.30 |
| 3개월 수익률 | 높을수록 좋음 | 0.20 |

- 12-1 모멘텀(`12-1Y Ratio`)은 이번에 신규 수집한 curated 팩터다(`collection/stock_base.py`).
  **주의:** 재수집 이전 스냅샷에는 이 컬럼이 없어 중립 50점으로 처리되며, 그동안 모멘텀은
  6M/3M이 실질적으로 주도한다. 주간 수집 워크플로가 한 번 돌면 자동 반영된다.

### ④ 안정성 (Stability) — 하드 필터 전용 (종합점수 미포함)
Altman Z 정신: 레버리지·유동성·커버리지·시장 리스크로 생존 확률을 근사한다.

| 팩터 | 방향 | 가중 |
|---|---|---|
| 순부채비율 (Net Debt to Equity) | 낮을수록 좋음(NEGATED) | 0.30 |
| 당좌비율 (Quick Ratio) | 높을수록 좋음 | 0.15 |
| 이자보상배율 (Interest Ratio) | 높을수록 좋음 | 0.15 |
| 현금흐름/부채 (Coverage Ratio) | 높을수록 좋음 | 0.15 |
| 1년 변동성 (1Y Volatility) `(반전)` | 낮을수록 좋음 | 0.25 |

- 순부채비율·설비투자비율은 음수/0이 정상값일 수 있어 1/x(RECIPROCAL) 대신 NEGATED(−x)를 쓴다.
- **금융 섹터 왜곡 처리:** 은행·보험은 레버리지가 구조적으로 높아 시장 전체 기준에서 일괄
  불리하다. 한국 종목의 Sector가 숫자 업종코드라(PROBLEMS #20) 이름 기반 예외가 불가능하므로,
  라벨에 의존하지 않는 **블렌드**로 완화한다:
  `안정성 필터 = 0.5 × 안정성(시장 전체) + 0.5 × 안정성(섹터 내)`. 하드 컷은 이 필터값으로 한다.

### ⑤ 재무건전성 (Health) — 가중 0.15
Piotroski F-Score 정신 + Novy-Marx 마진. **수집만 됐던 신규 재무 팩터 10종을 여기서 활용**한다.

| 구성 | 팩터 | 방향 | 가중 |
|---|---|---|---|
| 수익성 질 | ROA | 높을수록 | 0.10 |
| 현금흐름 질 | ARP (발생액) | 낮을수록(NEGATED) | 0.15 |
| 마진 | 매출총이익률 / 영업이익률 / 순이익률 | 높을수록 | 0.10 / 0.10 / 0.05 |
| 효율성 | 자산회전율 / 재고자산회전율 / 매출채권회전율 | 높을수록 | 0.10 / 0.075 / 0.075 |
| 부채 추세 | 부채 증가율 (Debt Growth) | 낮을수록(NEGATED) | 0.10 |
| 현금 완충 | 현금비율 (Cash Ratio) | 높을수록 | 0.10 |
| 투자 부담 | 설비투자/매출 (Capex to Revenue) | 낮을수록(NEGATED) | 0.05 |

- Piotroski 원전은 전년 대비 변화(Δ)를 보지만 현 스냅샷은 단일 시점이라 **수준값의 상대순위**로 근사한다.
- 유효법인세율(Effective Tax Rate)은 이월결손금·일회성 세액공제로 극단값이 과거 부실 신호일 수
  있어 **어떤 요인에도 넣지 않고** 화면·섹터 비교용으로만 채점한다.

### 종합점수 (QIP3 Score)
```
QIP3 Score = 가치성 × 0.35 + 성장성 × 0.25 + 모멘텀 × 0.25 + 재무건전성 × 0.15
```
안정성은 종합에 넣지 않고 별도 하드 필터로 쓴다(Greenblatt식 2단 선별).

## 3. 점수 컬럼 네이밍

기존 규칙(`QUANT.md` 2절)을 그대로 따른다. 모집단 태그 × 계열 태그:

- 요인: `QIP3 Value` / `QIP3 ValuePS` / `QIP3 ValueSS` / `QIP3 ValueSec…` / `QIP3 ValueInd…`
  (성장/모멘텀/안정성/건전성도 동일). 무접미사 = (퍼센타일 + 스탠다드)/2.
- 종합: `QIP3 Score` / `QIP3 ScoreSec` / `QIP3 ScoreInd`.
- 하드 필터: `QIP3 Stability Filter` (시장·섹터 블렌드, 단일 컬럼).

`snapshot_factors` 테이블은 동적 스키마라 이 컬럼들이 자동 추가된다. QIP3 점수 컬럼은
`CURATED_COLUMNS` 밖이므로 재점수 시 모집단 읽기 필터(`report_export._POPULATION_COLUMNS`)가
자동 차단한다 — PROBLEMS #31류 크래시가 구조적으로 재발하지 않는다.

## 4. 결측 처리

1. 모든 결측 팩터는 점수 50(중립). 데이터 존재 여부는 `{팩터}TF`로 기록.
2. 네이버 구조적 결측(내부자/기관 지분)은 QIP3에서 미사용해 원천 차단.
3. 재수집 이전 스냅샷에 **컬럼 자체가 없는** 신규 팩터(신규 10종·12-1 모멘텀)는
   파이프라인이 NaN 컬럼을 만들어(`_ensure_raw_factor_columns`) 중립 처리 → 재수집 시 자동 해소.

## 5. 선별 규칙 (`get_goodstock2`)

run 하나가 곧 시장 하나(KOSPI/KOSDAQ/NASDAQ/NYSE)이므로 "시장별 선별"이 run 단위로 구현된다.

1. **안정성 하드 컷:** `QIP3 Stability Filter`가 시장 내 하위 20%(20 백분위 이하)면 무조건 제외.
2. **신뢰도 가드:** `reliability > 50`. 결측 중립 50점이 누적돼 중위권에 위장 진입하는 것을 막는다.
3. **종합 랭킹:** 남은 기업을 `QIP3 Score` 내림차순으로 정렬해 **시장 전체 종목 수의 약 10%**를 선별.
   (생존자 수가 아니라 시장 전체 수 기준 10%이므로, 하드 컷으로 걸러진 만큼 실제 통과율은 더 낮다.)

상수는 `analysis/qip3_weights.py`에 있다: `STABILITY_CUT_QUANTILE=0.20`,
`SELECTION_RATIO=0.10`, `RELIABILITY_THRESHOLD=50`, 블렌드 가중 0.5/0.5.

## 6. 한계와 주의

- **시장 간 점수 비교 금지.** 네이버(한국)와 야후(미국)는 결측 팩터 구성이 다르다(PROBLEMS #9).
  선별·랭킹은 통화권(시장) 단위로만 유효하며, KR/US DB가 분리된 이유다.
- **소규모 섹터.** 섹터/산업 표본이 5개 미만이면 섹터 내 점수가 중립 50이 된다(PROBLEMS #22).
  안정성 필터의 섹터 블렌드도 이 경우 완만히 중앙으로 당겨질 뿐 특정 방향 편향은 없다.
- **12-1 모멘텀 재수집 전 중립.** 위 2절 ③ 참고.
- **단일 시점 근사.** 재무건전성은 Piotroski의 전년 대비 변화가 아니라 수준값 상대순위다.
- **정성 평가 미포함.** QIP3는 정량 점수만 산출한다. 뉴스·산업 전망 등 정성 조사는 별도 영역이다.
